Introduction

Usaruna is a multi-vendor e-commerce platform developed to support productive families in Saudi Arabia by providing a centralized digital marketplace for selling handmade and locally produced goods. The project relies heavily on software reuse by integrating existing frameworks, cloud services, database technologies, and authentication mechanisms instead of building every component from scratch. This approach reduced development effort while improving reliability, maintainability, and security.

My Assigned Concept

My assigned Software Engineering concept is Software Reuse. Software reuse is the practice of utilizing existing software assets such as frameworks, libraries, services, APIs, and architectural components to accelerate development and improve software quality. Rather than creating all functionality from the ground up, developers integrate proven technologies that have already been tested and optimized.

Reusable Technologies Used in Usaruna

React

Used as the primary frontend framework for building reusable user interface components and managing application state.

Vite

Used as the development and build tool to provide fast development, module loading, and production optimization.

Supabase

Used as a Backend-as-a-Service platform providing database management, authentication, storage, and API services.

Supabase Authentication

Reused for secure user registration, login, and session management without developing a custom authentication system.

Supabase Storage

Used for managing and storing product images in a scalable cloud storage environment.

PostgreSQL

Used through Supabase as the relational database engine for storing products, users, orders, reviews, and platform data.

PostgREST

Used to automatically expose database operations as RESTful APIs, reducing backend implementation effort.

Evidence of Software Reuse

  • React component architecture was used throughout the frontend application.
  • Supabase provided authentication, storage, and backend services.
  • PostgreSQL was reused as the database platform through Supabase.
  • PostgREST automatically generated API endpoints for database access.
  • Row-Level Security (RLS) policies provided built-in database-level access control.
  • Supabase Storage was reused to manage product image uploads and retrieval.

What I Learned

This assignment demonstrated the importance of software reuse in modern software engineering. I learned that successful systems are often built by integrating reliable technologies rather than creating every component independently. Through the Usaruna project, I observed how frameworks such as React and services such as Supabase significantly reduce development complexity while maintaining scalability, security, and performance.

Conclusion

Software reuse was a fundamental aspect of the Usaruna project. Existing technologies such as React, Supabase, PostgreSQL, PostgREST, JWT authentication, and Supabase Storage enabled the team to focus on implementing business requirements rather than rebuilding common infrastructure. The use of these reusable technologies contributed directly to the development of a secure, scalable, and maintainable e-commerce platform.
